What is GenomeSpace?

GenomeSpace is a cloud-based interoperability framework to support integrative genomics analysis through an easy-to-use Web interface. GenomeSpace provides access to a diverse range of bioinformatics tools, and bridges the gaps between the tools, making it easy to leverage the available analyses and visualizations in each of them. The tools retain their native look and feel, with GenomeSpace providing frictionless conduits between them through a lightweight interoperability layer. GenomeSpace does not perform any analyses itself; these are done within the member tools wherever they live – desktop, Web service, cloud, in-house server, etc. Rather, GenomeSpace provides tool selection and launch capabilities, and acts as a data highway automatically reformatting data as required when results move from the output of one tool to input for the next. The GVL GenomeSpace can be found at https://genomespace.genome.edu.au .

Prerequisites

GenomeSpace use a few dialogue box to communicate with NeCTAR cloud and if you have an “AdBlocker” on your system the dialogues will not be shown properly. So in case you have an AdBlocker on your system please disable it for GenomeSpace.genome.edu.au domain.

Registering into GenomeSpace

For registering into GenomeSPace you need to have a valid email address and follow the following steps:

  • Go to https://genomespace.genome.edu.au/ and click on the Register new GenomeSpace user link:
  • Enter your prefered username, password and the valid email address and click the Sign up Note: your username must be unique in GenomeSpace environment; otherwise you will receive an error. If everything goes right you will see the following page.
  • Go to the email you provided while registering and open the email from GenomeSpace Titled “GenomeSpace user registration” and follow the link in that email. From now on you can login to GenomeSpace by going to https://genomespace.genome.edu.au .
  • go to https://genomespace.genome.edu.au and enter your username and password you created in the previous steps. You will be logged in into GenomeSpace.

    In a few seconds you will be redirected to your Home page.

    In this page you can find the following items:
    1. your USERNAME on the top right corner
    2. The Menu bar
    3. The Application bar
    4. Your Home directory
    • The directory under the name Shared to “your username” contains any folders that has been shared to you through the GenomeSpace website.
    • The public directory is the directory which contains anything that has been made public to the cloud through the GenomeSpace website.

Mounting a swift container

(This session has written in a way to suit NeCTAR cloud for any other OpenStack based cloud storage please change the parameters as necessary)

NeCTAR object storage is a place that people with NeCTAR credentials can store their data reliably. Containers can be found under the Object Store link in NeCTAR’s dashboard. For browsing through available containers go to the NeCTAR dashboard at https://dashboard.rc.nectar.org.au and on the left hand side of the page you can find the Object Store. By selecting the container under the object store you can see all the available containers under your tenancy.

To mount an available container go to Connect menu bar and select Swift Container.

You will see a new page as follows:

To fill this form you need the following parameters:

  1. OpenStack EndPoint: It’s predefault value has set to connect to NeCTAR by default. https://keystone.rc.nectar.org.au:5000/v2.0/tokens
  2. NeCTAR UserName: Your username can be found at the top right corner of the NeCTAR dashboard as shown below:
  3. You API password: The API key is the key that applications can use to connect to NeCTAR on your behalf. To find your API key do the following steps:
    1. Login to your NeCTAR dashboard account.
    2. On the top right hand side of your Home click on the setting link
    3. Press the Reset Password button. (Warning: If you have already done this process for any other application you can use your old password. This process will reset your API key.)
  4. Your NeCTAR tenancy name. The Tenancy name has been assigned to your project by the NeCTAR administration process.
  5. The container’s name that you want to connect to.

Basic file manipulation

Under the containers directory you can do the basic file manipulation as follows:

  • Create Directory: For creating a directory under another directory right click on the source directory and select Create Subdirectory item. You will be asked for a name and in a few second your target directory will be created.
  • Uploading a file into a directory: Uploading a file is happening through drag and drop. First go to the directory you want to upload the file into and drag and drop the file you want to upload into the open area on the right hand side of Home directory. The effective area will be turned green.
  • Deleting a file: For deleting the file, right click on the selected file and select the “Delete” item. The file will be deleted in a few seconds.
  • Previewing the file: Under the right click menu you will find the preview item which will show the first 5000 bytes of a file.
  • Downloading a file: For downloading a file simply right click on the file and select download. Your download will be started in a few seconds.
  • Creating a public link: Right click on the file you want to get the public link and select the public link. The publick link will be shown to you in a few seconds. (Warning: The public link is available for 4 days.)
  • Create private link: The private link can be created by selecting the view file link item under the right click’s items.

Adding a Galaxy into your account:

PREREQUISITE: Please make sure you have registered on the Galaxy you want to add to GenomeSpace.

The latest GVL image is fully compatible with the GenomeSpace. The Galaxy launched under the GVL can be connected to the GenomeSpace as follows:

  1. From the Menu bar go to the manage menu and select Private Tool.

  2. From the opened window press the Add new button.

  3. In the new window fill the items as follow:
    • Give a name to your Galaxy
    • Give a description (Optional)
    • Tool provider GVL (Optional)
    • Base URL: http://[Glaxy-ip or DNS]/galaxy/tool_runner?tool_id=genomespace_importer
    • Parameter name: URL
    • Required: Ticked
    • Allow multiple files: Ticked
    • Multiple file Delimiter: ,
    • Select the files’ types that you want your galaxy to work on
    • Upload an image as an Icon for Galaxy (Optional)
    • Press the save button. In a few seconds your Galaxy instance will be added to the Application bar.

    A sample page can be seen in the following image.

  4. Launching the added Galaxy from GenomeSpace: From the GenomeSpace click on the Arrow on the right side of your Galaxy application in the Application bar and select launch.Your galaxy will be opened in a new window.

    (Note: Your browser may block the pop-up. Allow the pop-up accordingly).
  5. From the opened Galaxy login and under your username go to the preferences options

    and select the Manage OpenIDs links:

  6. From the associate more OpenID select the GenomeSpace and press login. Your GenomeSpace will be appear as link on the top as a URL.
From now on your Galaxy can talk to the GenomeSpace under your UserName.

Sending/Receiving a file into/from Galaxy:

PREREQUISITE: Please make sure you have connected your Galaxy to GenomeSpace first (How to).

  • Sending A file
    • From your Galaxy go to Get Data and select GenomeSpace Importer (Please make sure you are logged into GenomeSpace). You will see your genomespace home page in a few seconds. Select the file you want to send into your Galaxy and press the Send to Galaxy button. A new job will be created in your Galaxy and your file will be in the Galaxy by the completion of the job.
    • From GenomeSpcae You can also send a file into Galaxy by from The GenomeSpace home page. Simply drag and drop the file into your Galaxy from your GenomeSpace Home page.
  • Receiving A file

    Under each file click on the store icon and choose send to GenomeSpace. A dialogue box will be opened. Choose the directory to store the file and enter the name of the file to store. If you do not provide the file name file will be stored as “display”. By click send button the file will be send to GenomeSpace and the dialogue box will be closed on success.
